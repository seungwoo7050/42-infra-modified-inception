# Inception v0.2.0 - Nginx Reverse Proxy + Redis

## Goal
- 앱 앞단에 리버스 프록시(Nginx)를 추가하고, 캐시/세션 용도로 Redis를 도입한다.

## Scope
- Nginx:
  - TLS 종료(또는 개발용 self-signed 설정) 및 기본 라우팅
  - 정적 파일 서빙(필요 시)과 프록시 패스 설정
  - 로그 포매팅 및 기본 보안 헤더 적용
- Redis:
  - 세션 저장 또는 캐시 계층(예: 빈번한 쿼리 결과 캐싱) 역할
  - compose에서 Redis 서비스 추가 및 환경변수로 설정

## Completion Criteria
- `docker-compose.yml` 또는 `docker-compose.v0.2.yml`에 `nginx`와 `redis` 서비스 정의
- Nginx가 앱으로 프록시되어 HTTPS(또는 HTTP) 요청을 정상 전달함
- 앱이 Redis를 사용해 캐시/세션을 저장하는 간단 예시(동작 확인 스크립트 포함)
- 관련 문서: `design/v0.2.0-nginx-redis-stack.md` 및 `CLONE_GUIDE.md`에 실행 방법 추가

## Design Notes
- 운영과 개발을 분리한 Nginx 설정 권장(개발은 간단한 HTTP, 운영은 TLS + HSTS 등)
- Redis는 persistence 옵션(AOF/RDB)을 필요에 따라 설정, 기본 개발 환경은 non-persistent 권장
- 세션 사용 시 보안: Redis 접근은 내부 네트워크로 한정, 비밀번호(또는 ACL) 설정 권장

## Implementation Steps
1. `nginx/` 폴더에 개발용과 운영용 conf 템플릿 작성
2. `docker-compose`에 `nginx`와 `redis` 서비스 추가
3. 앱(현재 Spring Boot)에서 Redis(예: Spring Session 또는 Jedis/Lettuce) 연동 예제 추가
4. 로컬 검증: `curl`로 Nginx → app 프록시 동작 확인, Redis 캐시 작동 확인

## Caveats
- 로컬에서 TLS를 실험할 경우 브라우저/OS 신뢰 문제 고려
- Redis를 캐시로 쓰는 경우 캐시 무효화 전략을 반드시 설계해야 함
